## 迭代修復過程

### Bug 1: URL 為空
- `patent_search_v2.py` 搜索階段未正確構建 Google Patents URL
- 候選專利 URL 均為空字串
- **修復**: 使用完整域名 `https://patents.google.com/patent/{id}/en`

### Bug 2: 標題顯示 "Patent"
- `patent_extract_v3.py` 使用 `document.querySelector('h1')` 提取標題
- 結果只返回 "Patent" 而非完整標題
- **修復**: 改用 `page.title()`，標題格式為 "EP4733370A1 - Liquid-crystal medium - Google Patents"

### Bug 3: Filing date 全部未提取
- `patent_extract_v5.py` 搜索 "Filed:" 前綴，但頁面無此格式
- **修復**: 使用 meta dates + `[data-timeline-field="filingDate"]` timeline 元素

### Bug 4: Claim1 不完整
- EP/WO 專利頁面較長，預設滾動不足
- **修復**: 滾動至頁面 80% 位置，增加等待時間

### Bug 5: JS evaluate 正則語法錯誤
- `patent_extract_v4.py` 中 JS 正則含未轉義括號
- **修復**: 改用純 Python 正則從 `page.inner_text('body')` 提取

### Bug 6: phys_params 為 list 而非 dict
- `generate_report_and_push.py` 假設 phys_params 為 dict，實際為 list
- **修復**: 兼容 `isinstance(params, (dict, list))` 兩種格式

## 數據質量清理

### False Positive 移除
- **US20250361444A1**: neg=2, pos=6 — 化合物專利非液晶介質專利，Claim1 不含 "liquid crystal medium"
- 判定規則: neg_count <= pos_count 且 Claim1 不含 LC medium 關鍵字 → 移除

### A1/B2 去重
- US20250207032A1 和 US12612551B2 為同一專利
- 保留 B2（已授權），移除 A1（申請案）

### 分子結構亂碼
- 多個 US 專利返回 1221 個單字元 token（f, o, r, m, u, l, a）
- 原因: 正則匹配 `formula` 單詞時逐字拆解
- 過濾規則: `len(m.strip()) > 3 and not re.match(r'^[a-z]$', m.strip())`

### US 專利摘要 UI 污染
- 部分美國專利摘要區段被 Google Patents UI 控件文本覆蓋
- 摘要只顯示 "Claims All Any Exact" 而非實際技術內容
- 報告中需標註此問題或從其他段落重建摘要

## 關鍵教訓

1. **模組化腳本 > 單體 E2E**: 調試效率顯著提升，每個 bug 可獨立定位修復
2. **前台執行 > 背景執行**: `python3 -u` + `PYTHONUNBUFFERED=1` 確保即時輸出
3. **清理階段不可省略**: 原始數據必定含 false positive 和亂碼，需專門清理腳本
4. **分子結構提取是薄弱環節**: 正則方法不夠穩定，單字元過濾是必要的後處理
