## 發現的問題

### 問題 1: EP 專利 description 提取為空
- `extract_patent_sections()` 的 `r'Description\n([\s\S]{200,}?)\nClaims'` 正則對 EP 頁面無效
- EP 頁面的 Description 與 Claims 之間的 DOM 標記可能不同
- 需使用深度提取（inner_text 200K+ chars）才能獲取完整 description → 見陷阱 20
- **影響**: neg/pos 自動判定失敗 + 實施例缺失 + 技術要點「實施方式」維度無法判斷

### 問題 2: neg/pos 判定需回退策略
- 當 description 為空時，應改從 abstract + background + summary 進行關鍵字計數
- EP4538349A1 的 Background 含 "positive dielectric anisotropy"，pos_count≥1 即可判定
- **解決方案**: 寫入陷阱 27，neg/pos 計數回退邏輯

### 問題 3: contrast 搜索命中正介電專利
- EP4538349A1 因 Background 含 "high contrast gaming displays" 被 contrast 搜索命中
- 但其為正介電 FFS 模式專利，與負介電液晶研究目標不符
- 這不是 bug，是搜索策略的正常覆蓋範圍，應由 neg/pos 過濾排除
