# 代码审查

代码审查的双向规范：如何请求审查和如何响应审查反馈。

## 何时使用

- 完成功能或修复后需要审查
- 收到审查反馈需要评估和实施

## 子流程

### 请求代码审查（requesting-code-review）

完成任务、实现重要功能或合并前使用，用于验证工作成果是否符合要求。

**核心原则：** 早审查，勤审查。

**详细参考：** `references/requesting-code-review.md`

### 接收代码审查（receiving-code-review）

收到代码审查反馈后、实施建议之前使用——需要技术严谨性和验证，而非敷衍附和或盲目执行。

**核心原则：** 先验证再实施。先提问再假设。技术正确性优先于社交舒适度。

**详细参考：** `references/receiving-code-review.md`
